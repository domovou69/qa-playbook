# reports/

Place your Mochawesome JSON report here before running the scanner.

## Quick start

```bash
# From the coverage-audit/ directory
npm install

# Source-only scan (no runtime data)
npm run scan -- --tests "../path/to/tests/**/*.spec.ts" --cwd ..

# Full scan with runtime data
npm run scan -- \
  --tests  "../path/to/tests/**/*.spec.ts" \
  --report reports/mochawesome.json \
  --cwd    ..
```

Output is written to `reports/enriched-metadata.json`.

## Mochawesome report format

The scanner expects the standard Mochawesome JSON format (single file, not the
multi-file variant). If you use `mochawesome-merge`, run the merge step first:

```bash
npx mochawesome-merge reports/mochawesome-*.json -o reports/mochawesome.json
```

## Files in this directory

| File | Description |
|------|-------------|
| `mochawesome.json` | Your Mochawesome report — **add before scanning** |
| `enriched-metadata.json` | Generated by scan.ts — passed to Claude for audit |

Both JSON files are gitignored (see `.gitignore`).
